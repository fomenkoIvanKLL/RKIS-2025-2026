# Тест-кейсы для TodoList Application

## Введение
Данный документ содержит набор тест-кейсов для проверки функциональности консольного приложения TodoList.  
Все скриншоты выполнения тестов находятся в папке `Screenshots`.

---

## 1. Пользователи и авторизация

### TC-01 Создание нового пользователя
**Описание:** Проверка успешного создания нового профиля.  
**Предусловия:** Приложение запущено, нет активного пользователя.  
**Последовательность действий:**
1. При запросе "Войти в существующий профиль? [y/n]" ввести `n`.
2. Ввести логин: `testuser`.
3. Ввести пароль: `pass123`.
4. Ввести имя: `Тест`.
5. Ввести фамилию: `Тестов`.
6. Ввести год рождения: `2000`.
**Ожидаемый результат:**  
- Сообщение "Профиль создан! Добро пожаловать, Тест Тестов!"  
- Профиль сохраняется в файле `data/profiles.csv`.  
**Скриншоты:**  
- `Screenshots/TC-01-step1.png`  

### TC-02 Авторизация существующего пользователя
**Описание:** Проверка входа в созданный ранее профиль.  
**Предусловия:** Существует пользователь с логином `testuser` и паролем `pass123`.  
**Последовательность действий:**
1. При запросе выбрать `y`.
2. Ввести логин: `testuser`.
3. Ввести пароль: `pass123`.
**Ожидаемый результат:**  
- Сообщение "Добро пожаловать, Тест Тестов!"  
- `CurrentProfileId` устанавливается для этого пользователя.  
**Скриншоты:** `Screenshots/TC-02-login.png`

### TC-03 Ошибка авторизации (неверный пароль)
**Описание:** Попытка входа с неверным паролем.  
**Предусловия:** Пользователь `testuser` существует.  
**Последовательность действий:**
1. При запросе выбрать `y`.
2. Ввести логин: `testuser`.
3. Ввести пароль: `wrongpass`.

### TC-04 Выход из профиля

**Последовательность действий:**
1. `add Это очень длинный текст задачи, который должен быть обрезан в списке`

- Задача добавлена с текстом, содержащим переводы строк.

### TC-11 Добавление задачи со служебными символами

**Ожидаемый результат:**
